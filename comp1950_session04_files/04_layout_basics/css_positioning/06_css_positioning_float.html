Box Model Floating

When a block tag is floated, it will collapse to the size of its contents (similar to an inline tag). It is therefore usually best to assign width and/or height to floated tags.

Nested Floating

Generally it is best to avoid floating elements inside of other floated elements, as you may find the deeper the nesting goes, the more unpredictable the results. However, it is useful to be aware of an interesting characteristic of floated parent/child relationships.

In the following example, a burlywood colored parent is not floated, but it's lighter beige colored child is. The parent div has a black border, which you can still see as a line along the top, but note how the height of the parent has collapsed.

Nested Floating Solutions

There are 3 options for handling parent containers that collapse due to floated children:

  1. Using a technique described earlier on this page, you can add a block tag with clear:both; as the last child of the parent.

  2. Or you can apply overflow:auto; to the collapsing parent
