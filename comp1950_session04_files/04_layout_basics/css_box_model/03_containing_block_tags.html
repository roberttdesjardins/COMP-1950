CSS box model: containing block tags

A common 'rule of thumb' for HTML is that block tags don't usually contain other block tags as children. Sometimes this rule must be broken, for example, when using the generic div tag to contain content. Doing this is a structurally valid use of HTML, but does have some odd 'behavioral' effects.

In the examples below... block parents are colored black, block children are highlighted yellow

Top of parent container...

  1. with no margin or padding on either parent or child, note how the child is positioned at the top of the parent container. use your browser diagnostic tool to identify the current margin of the child. Note how the margin appears to 'escape' the parent.

    Block Tag Child


  2. now a small padding-top is added to the parent. note how the child's margin now asserts itself within the parent container!

    Block Tag Child


  3. remember you can always 'regain control' by altering the margins of the child, for example, by setting margin-top to be zero px.

    Block Tag Child


Bottom of parent container...

  1. once again, neither parent nor child are being assigned margins/padding. this time however, the parent container's height is NOT set, so it is exactly as tall as it needs to be for its child

    Block Tag Child


  2. this time, the parent container's padding-bottom has been set to one px, note how the same effect on the child's margins occurs

    Block Tag Child


  3. as before, fix these issues by altering the child's margin-bottom

    Block Tag Child
