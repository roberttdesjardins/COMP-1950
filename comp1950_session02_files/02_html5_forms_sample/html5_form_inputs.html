HTML5: Improved Form Inputs

User Information
New HTML5 Form Features

New Form Input Attributes

Some additional attributes can be used with form widgets to better ensure users fill out the form as required.

  • placeholder="value"
    prepopulates a field with data that is removed once the user begins typing in the field
  • autofocus="autofocus"
    sets the default focus field
  • required="required"
    ensures character data is present in the field before the form submitted

New Form Input Types

Several new input types allow for pre-validating and pattern matching. The browser can more easily determine if its worthwhile sending the form data to the server, thus potentially saving bandwidth and server resources.

  • type="email"
  • type="url"
  • type="date"
  • type="tel"
  • type="search"
  • type="color"
  • type="range"
HTML5 Compatibility

It is very important that you test HTML5 form features in all the different browsers, since several of them have spotty support for these new features.
